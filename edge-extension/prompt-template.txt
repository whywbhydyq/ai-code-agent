你好，请在后续对话中，当涉及代码文件的创建、修改或删除时，严格按照以下格式输出。

## 格式规范

所有代码操作指令必须包裹在 ```agent-action 代码块中，内容是标准 JSON。

### 1. 创建新文件 或 完整替换现有文件（推荐）

```agent-action
{
  "action": "write",
  "file": "相对路径/文件名.扩展名",
  "content": "完整的文件内容..."
}
```

### 2. 局部修改现有文件（仅用于超过 100 行的大文件，且只改 1-2 处）

```agent-action
{
  "action": "patch",
  "file": "相对路径/文件名.扩展名",
  "patches": [
    {
      "find": "要替换的代码（2-3行简单代码）",
      "replace": "替换后的代码"
    }
  ]
}
```

### 3. 删除文件

```agent-action
{
  "action": "delete",
  "file": "相对路径/文件名.扩展名"
}
```

## Patch 规则（非常重要）

### find 字段的写法：
- find 只需要包含 2-3 行能唯一定位的简单代码
- 绝对不要在 find 中包含 f-string、三引号、正则表达式、复杂转义字符
- 如果目标代码含有引号或反斜杠，改用 after+insert 或直接用 write 模式

### 推荐用 after+insert 代替 find+replace：
```agent-action
{
  "action": "patch",
  "file": "example.py",
  "patches": [
    {
      "after": "def my_function():",
      "insert": "    print('new line')"
    }
  ]
}
```

### 推荐用 delete 删除行：
```agent-action
{
  "action": "patch",
  "file": "example.py",
  "patches": [
    {
      "delete": "    old_line_to_remove()"
    }
  ]
}
```

## 选择 write 还是 patch 的标准

- 文件 < 100 行 → 直接用 write，输出完整内容
- 文件 > 100 行且只改 1-2 处简单行 → 用 patch
- 文件改 3 处以上 → 必须用 write，禁止用 patch
- 遇到含引号、三引号、正则、转义字符的代码 → 绝对用 write，不要用 patch
- 不确定 find 能不能匹配到 → 直接用 write，不要赌

## 禁止的行为

1. 绝对不要在 content 中使用省略写法（如 // ... existing code ...）
2. write 模式必须输出完整文件内容，一行都不能省
3. find 字段不要包含复杂转义字符（引号、反斜杠、三引号、正则）
4. 文件路径使用 / 分隔
5. patch 如果不确定能匹配，必须改用 write，不能给出可能失败的 patch
6. 同一文件改 3 处以上，必须用 write 输出完整文件

## 协作规则

1. 如果我贴了当前代码给你看，以我贴的为准，不要依赖你的记忆
2. 改完后简要说明改了哪几行、为什么改，方便我核对
3. 如果一个修改可能影响其他文件，主动告诉我还需要改哪里

请确认你理解了以上格式要求。
