你好，请在后续对话中，当涉及代码文件的创建、修改或删除时，严格按照以下格式输出。

## 格式规范

所有代码操作指令必须包裹在 ```agent-action 代码块中，内容是标准 JSON。

### 1. 创建新文件 或 完整替换现有文件

```agent-action
{
  "action": "write",
  "file": "相对路径/文件名.扩展名",
  "content": "完整的文件内容..."
}
```

### 2. 局部修改现有文件（推荐用于超过 50 行的大文件）

```agent-action
{
  "action": "patch",
  "file": "相对路径/文件名.扩展名",
  "patches": [
    {
      "find": "要被替换的原始代码（精确复制，包含前后至少 2 行上下文）",
      "replace": "替换后的完整代码"
    },
    {
      "after": "在这行代码之后插入",
      "insert": "要插入的新代码"
    },
    {
      "delete": "要删除的这行代码"
    }
  ]
}
```

### 3. 删除文件

```agent-action
{
  "action": "delete",
  "file": "相对路径/文件名.扩展名"
}
```

## 重要规则

1. 绝对不要在 content 中使用省略写法（如 // ... existing code ...）
2. 如果用 write 模式，必须输出完整文件内容，一行都不能省
3. patch 的 find 字段至少包含目标代码前后各 1-2 行上下文
4. 一次回答可以输出多个 agent-action 代码块
5. 文件路径使用 / 分隔
6. 小文件（< 50 行）直接用 write，大文件用 patch

请确认你理解了以上格式要求。
