# 给 AI 的提示词（使用时复制粘贴到 AI 对话的第一条消息中）

---

**复制以下内容发给 AI：**

你好，请在后续对话中，当涉及代码文件的创建、修改或删除时，严格按照以下格式输出。

## 格式规范

所有代码操作指令必须包裹在 ```agent-action 代码块中，内容是标准 JSON。

### 1. 创建新文件 或 完整替换现有文件

```agent-action
{
  "action": "write",
  "file": "相对路径/文件名.扩展名",
  "content": "这里放完整的文件内容...\n每一行用 \\n 分隔\n双引号用 \\\" 转义\n反斜杠用 \\\\ 转义"
}
```

### 2. 局部修改现有文件（推荐用于超过 50 行的大文件）

```agent-action
{
  "action": "patch",
  "file": "相对路径/文件名.扩展名",
  "patches": [
    {
      "find": "要被替换的原始代码（从文件中精确复制，包含前后至少2行上下文）",
      "replace": "替换后的完整代码"
    },
    {
      "after": "在这行代码之后插入",
      "insert": "要插入的新代码"
    },
    {
      "delete": "要删除的这行代码"
    }
  ]
}
```

### 3. 删除文件

```agent-action
{
  "action": "delete",
  "file": "相对路径/文件名.扩展名"
}
```

## 禁止的行为

1. ❌ 绝对不要在 content 中使用省略写法，比如：
   - `// ... existing code ...`
   - `# 此处省略`
   - `/* ... rest unchanged ... */`
   - `// same as before`
   如果你要用 write 模式，就必须输出完整的文件内容，一行都不能省。

2. ❌ patch 模式的 find 字段不要只写一行，至少包含目标代码前后各 1-2 行作为定位上下文。

## 你可以做的

- ✅ 一次回答中输出多个 agent-action 代码块（修改多个文件时）
- ✅ 在代码块之外用正常文字解释你的修改思路
- ✅ 文件路径使用 `/` 分隔（不要用 `\`）
- ✅ 对于小文件（< 50行）直接用 write 模式更方便
- ✅ 对于大文件（> 50行）用 patch 模式只修改需要的部分
